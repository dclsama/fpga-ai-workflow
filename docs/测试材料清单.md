## 四、失败案例与改进记录

### 案例 1：GCD 死循环（JavaScript 模拟器）

| 项目 | 内容 |
|------|------|
| **问题描述** | GCD Demo 3 在 Web GUI 中执行时出现死循环，无法到达 HALT |
| **根因** | JavaScript 模拟器中 `executeOneInstruction()` 使用 `rd = byte3 & 0x07` 作为目标寄存器，但 LDI/LD/IN 指令的目标寄存器分别位于 byte2 或 byte1 |
| **具体表现** | `LDI R1, 18` 错误写入 R0，导致 R1 始终为 0，程序逻辑错误 |
| **发现方式** | 添加 `console.log` 逐步追踪 30 步执行轨迹，AI 逐指令分析寄存器状态 |
| **修复方案** | 在 LDI(0x12): `rd = byte2 & 0x07`; LD(0x10): `rd = byte2 & 0x07`; IN(0x50): `rd = byte1 & 0x07` |
| **修复后结果** | GCD 程序正常输出 0x06 |
| **相关提交** | `ea4e038`, `22c5a94`, `3b89e5e` |

### 案例 2：ALU 标志更新缺失（汇编器）

| 项目 | 内容 |
|------|------|
| **问题描述** | Demo 程序中 JZ/JC 条件跳转不生效 |
| **根因** | 汇编器未设置 byte3 的 bit[3]，导致 ALU reg-reg 指令不更新 Z/C 标志 |
| **修复方案** | 在 ALU_REG3 和 NOT 指令编码中设置 `byte3 = rd | 0x08` |
| **相关提交** | `fd2f106` |

## 六、测试执行方式

### Verilog 仿真（模块级）

```bash
# 单个模块完整仿真
./scripts/sim.sh sim leg_alu
./scripts/sim.sh sim leg_decoder
./scripts/sim.sh sim leg_core

# 查看波形
./scripts/sim.sh wave leg_core

# 导出 VCD 用于深度分析
./scripts/sim.sh export leg_core

# 解析日志提取错误
./scripts/sim.sh parse leg_core
```

### Web GUI 模拟器（程序级）

1. 浏览器打开 `leg_gui.html`
2. 从 Demo 下拉框选择程序
3. 点击 Assemble 编译
4. 点击 Step 逐步执行，或 Run 全速运行
5. 观察右侧状态面板和输出控制台
